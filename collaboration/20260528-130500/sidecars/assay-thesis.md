# Sidecar — The assay thesis (depth)

> **Assay** - a precise test of what something is genuinely made of; an *assayer* certifies the true composition of a metal. Here, the *assay pathways* test what a legal-AI claim is really made of - grounded citations, reproducible scores, attested runs: the honest measurement the satire only performs.


Backs `collaboration-spine.md` §3. Optional depth; read only if the thesis made you want more.

## The one-sentence thesis

The gap between *marketing* legal AI and *trustworthy* legal AI is not model quality; it is whether a claim can be **checked** — and checkability is a commons, owned by neither project.

## Four load-bearing properties

1. **Verify-first grounding.** Deterministic lookups against the public commons run before any model speaks. The single most reliable source of hallucinated citations is a model's latent recall; the cure is to not ask it to recall in the first place.

2. **Multi-model reconciliation.** Run the same question across models as separate, individually-attested runs; reconcile narratively over provenance rather than collapsing into one confident voice.

3. **Provenance / attestation.** Hashed prompts and outputs, signed identity, offline-verifiable. This is what makes "who-ran / who-attested / who-authored" visible — the structural opposite of a self-administered panel.

4. **Honest convergence (recursion read as truth).** Re-run bounded against anchored authority until typed error is low; never re-open latent recall once anchored; attest each iteration; bound depth and budget.

## Why neither project should "own" it

If the assay for benchmark theater were proprietary, it would be just another moat — and the satire would win. The thesis is therefore offered as a commons: a shared vocabulary (the four `assay-pathways/` stubs), built on a shared substrate (the public legal-data commons), creditable and forkable by anyone.

## Honest limits

Several of these capabilities are early-stage in the Pathways project itself (for example, the convergence loop is specified more completely than it is shipped). The asks in `collaboration-pathways/test/HYPOTHESES.md` are honest experiments to close that gap in the open, not claims that it is already closed.
